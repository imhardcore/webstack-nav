## 二、网址管理（sites.json）

### 数据结构

```json
{
  "sites": [
    {
      "id": "chatgpt",
      "title": "ChatGPT",
      "url": "https://chat.openai.com/",
      "description": "OpenAI 开发的对话式 AI 工具",
      "logo": "/webstack-nav/images/logos/chatgpt.png",
      "category": "AI",
      "group": "nav"
    }
  ]
}
```

### 字段说明

| 字段 | 类型 | 必填 | 说明 |
|:---|:---|:---|:---|
| id | string | 是 | 唯一标识符，英文小写，如 `chatgpt` |
| title | string | 是 | 网站显示名称 |
| url | string | 是 | 网站链接地址 |
| description | string | 是 | 网站描述，显示在卡片上 |
| logo | string | 否 | 图标图片路径。通过 CMS 上传后会自动存入 `static/images/logos/`，路径格式为 `/webstack-nav/images/logos/xxx.png` |
| category | string | 是 | 所属分类，必须与 `categories.json` 中子分类的 `id` 一致。当前可选值：`AI`、`GFW`、`SillyTavern`、`设计资源`、`影视资源` |
| group | string | 是 | 所属分类组，当前固定为 `nav` |

### 关联关系

- `category` 字段的值必须存在于 `categories.json` → `groups[0].categories[*].id`
- `group` 字段的值必须存在于 `categories.json` → `groups[*].id`

| 字段 | 类型 | 必填 | 说明 |

| 字段 | 类型 | 必填 | 说明 |

| id | string | 是 | 分类标识符，与 `sites.json` 中的 `category` 字段对应。修改后需同步更新所有相关网址的 `category` 字段 |

## 五、Logo 配置（logo.json）

### 数据结构

```json
{
  "logo": {
    "type": "text",
    "text": "My Nav",
    "accentText": "My",
    "image": "",
    "imageAlt": "我的导航",
    "imageWidth": 32,
    "imageHeight": 32
  }
}
```

### 字段说明

| 字段 | 类型 | 必填 | 说明 |
|:---|:---|:---|:---|
| type | string | 是 | 标识类型：`text`（文字标识）或 `image`（图片标识） |
| text | string | 否 | 完整标识文字。如 `我的导航`。显示在顶部工具栏左侧和 CMS 登录页标题 |
| accentText | string | 否 | 高亮部分文字。如 `我的`。取首字作为 RGB 渐变图标 |
| image | string | 否 | 图片路径。`type` 为 `image` 时生效 |
| imageAlt | string | 否 | 图片替代文字 |
| imageWidth | number | 否 | 图片宽度（像素） |
| imageHeight | number | 否 | 图片高度（像素） |

### 当前数据状态

```json
{
  "logo": {
    "type": "text",
    "text": "My Nav",
    "accentText": "My",
    "image": "",
    "imageAlt": "我的导航",
    "imageWidth": 32,
    "imageHeight": 32
  }
}
```

### 修改注意事项

- `type` 为 `text` 时，优先使用 `accentText` 首字作为图标，若未填写则使用 `text` 首字
- `type` 为 `image` 时，需填写 `image` 路径，并建议同时设置 `imageWidth` 和 `imageHeight`
- 修改后 CMS 登录页标题和图标会自动更新

## 八、修改建议

### 如果你要添加新网址

1. 先确认分类已存在（在**分类管理**中检查）
2. 在**网址管理**中添加，注意 `category` 字段必须与分类 id 完全一致
3. 图标可选：上传图片或留空（留空时前端会显示默认占位图标）

### 如果你要修改分类

1. 修改分类 id 后，必须同步修改所有对应网址的 `category` 字段
2. 建议在 CMS 中批量编辑，避免手动修改 JSON 导致格式错误
3. 分类 id 建议保持简洁英文，如 `ai`、`movie`、`tool`

### 如果你要清空数据重新开始

1. **网址管理**：进入编辑界面，逐个删除或清空 `sites` 数组
2. **分类管理**：修改 `groups[0].categories` 数组
3. **快捷链接**：修改 `quickLinks` 数组
4. **Logo 配置**：修改 `logo` 对象字段
